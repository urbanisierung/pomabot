# Audit Logs

This directory contains CSV audit logs generated by PomaBot.

## Log Format

Each log file is named `audit-YYYY-MM-DD.csv` and contains the following columns:

- **timestamp** - ISO 8601 timestamp
- **event** - Event type (SYSTEM_START, MARKET_EVALUATED, TRADE_OPPORTUNITY, etc.)
- **marketId** - Polymarket market ID (if applicable)
- **marketQuestion** - Market question text
- **action** - Action taken (PASSED, FAILED, BUY, SELL, etc.)
- **details** - Event details and rationale
- **belief** - Belief midpoint (0-100)
- **edge** - Edge percentage
- **amount** - Trade amount in USDC
- **pnl** - Profit/Loss amount

## Event Types

- **SYSTEM_START** - Bot startup
- **SYSTEM_STOP** - Bot shutdown
- **MARKET_EVALUATED** - Market evaluation result
- **TRADE_OPPORTUNITY** - Trade opportunity detected
- **TRADE_EXECUTED** - Trade executed
- **POSITION_CLOSED** - Position closed
- **ERROR** - Error occurred
- **DAILY_SUMMARY** - Daily summary report

## Git Persistence (Optional)

To persist audit logs to Git:

```bash
git add audit-logs/
git commit -m "chore: update audit logs $(date +%Y-%m-%d)"
git push origin main
```

Consider setting up a scheduled job (cron) to automatically commit logs daily.

## External Logging

If `LOGTAIL_TOKEN` is set, logs are also shipped to Logtail (Better Stack) for:
- Real-time search and filtering
- Alerts and notifications
- Historical analysis
- Dashboard visualization

See DEPLOYMENT.md for setup instructions.
